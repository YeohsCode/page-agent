# PageAgent 数据流与模块通信

## 🔄 一次完整任务的数据流

以用户输入 **"点击登录按钮"** 为例，展示完整的数据流动：

```
用户输入: "点击登录按钮"
        │
        ▼
┌─────────────────────────────────────────┐
│  PageAgent.execute("点击登录按钮")       │  ← packages/page-agent
│  (内部调用 PageAgentCore.execute)        │
└────────────────┬────────────────────────┘
                 │
        ┌────────▼────────┐
        │  Step 0 开始     │  ← packages/core
        └────────┬────────┘
                 │
   ══════════════╪══════════════  观察阶段
                 │
        ┌────────▼─────────────────────┐
        │  pageController.getBrowserState()  │  ← packages/page-controller
        │  1. 遍历 DOM                      │
        │  2. 提取交互元素                   │
        │  3. 生成简化 HTML                  │
        │  4. 构建 SelectorMap               │
        └────────┬─────────────────────┘
                 │
                 │ 返回 BrowserState:
                 │ {
                 │   url: "https://example.com",
                 │   header: "Page info: ...",
                 │   content: "[0]<button>登录</button>\n[1]<input>...",
                 │   footer: "[End of page]"
                 │ }
                 │
   ══════════════╪══════════════  思考阶段
                 │
        ┌────────▼─────────────────────┐
        │  组装 Prompt                      │  ← packages/core
        │  system_prompt + user_prompt       │
        │  + 历史记录 + 浏览器状态           │
        └────────┬─────────────────────┘
                 │
        ┌────────▼─────────────────────┐
        │  LLM.invoke(messages, tools)      │  ← packages/llms
        │  1. Zod Schema → OpenAI Tool 格式  │
        │  2. 模型适配补丁                   │
        │  3. HTTP POST /chat/completions    │
        │  4. 解析响应                       │
        │  5. 验证工具参数                   │
        └────────┬─────────────────────┘
                 │
                 │ LLM 返回:
                 │ {
                 │   "evaluation_previous_goal": "首次执行",
                 │   "memory": "需要找到并点击登录按钮",
                 │   "next_goal": "点击索引 0 的登录按钮",
                 │   "action": {
                 │     "click_element_by_index": { "index": 0 }
                 │   }
                 │ }
                 │
   ══════════════╪══════════════  行动阶段
                 │
        ┌────────▼─────────────────────┐
        │  MacroTool.execute()              │  ← packages/core
        │  找到对应工具并执行                │
        └────────┬─────────────────────┘
                 │
        ┌────────▼─────────────────────┐
        │  pageController.clickElement(0)    │  ← packages/page-controller
        │  1. 从 SelectorMap 找到真实元素    │
        │  2. 滚动到可见                     │
        │  3. 移动虚拟指针                   │
        │  4. 模拟完整鼠标事件               │
        └────────┬─────────────────────┘
                 │
                 │ 返回: "✅ Clicked element (登录)"
                 │
        ┌────────▼─────────────────────┐
        │  记录历史、触发事件               │  ← packages/core
        │  → historychange 事件             │
        │  → Panel 更新显示                 │  ← packages/ui
        └────────┬─────────────────────┘
                 │
        ┌────────▼────────┐
        │  下一步或结束    │
        └─────────────────┘
```

## 📡 模块间通信方式

### 1. PageAgentCore ↔ PageController（方法调用）

所有通信通过异步方法调用：

```typescript
// Agent 请求页面状态
const state = await this.pageController.getBrowserState()

// Agent 请求执行操作
await this.pageController.clickElement(index)
await this.pageController.inputText(index, text)
await this.pageController.scroll({ down: true, numPages: 1 })
```

### 2. PageAgentCore ↔ LLM（HTTP API）

通过 OpenAI 兼容的 REST API 通信：

```
POST {baseURL}/chat/completions
Headers: Authorization: Bearer {apiKey}
Body: { model, messages, tools, tool_choice, temperature }
```

### 3. PageAgentCore ↔ Panel（事件系统）

通过 `EventTarget` 事件发布/订阅：

```typescript
// Agent 发出事件
this.dispatchEvent(new Event('statuschange'))
this.dispatchEvent(new Event('historychange'))
this.dispatchEvent(new CustomEvent('activity', { detail: activity }))

// Panel 监听事件
agent.addEventListener('statuschange', handler)
agent.addEventListener('historychange', handler)
agent.addEventListener('activity', handler)
```

### 4. MCP ↔ Extension Hub（WebSocket）

```
MCP Server ←→ WebSocket ←→ Hub Tab（Chrome 扩展页面）
消息格式: JSON { type: 'execute'|'stop'|'result'|'error', ... }
```

## 📊 信息流分类

项目中有两种信息流，用途不同：

### 1. 历史事件（History Events）— 持久化

- **存储在** `agent.history` 数组中
- **包含在** LLM 上下文中（跨步骤记忆）
- **类型**：step（步骤）、observation（观察）、user_takeover（用户接管）、retry（重试）、error（错误）

### 2. 活动事件（Activity Events）— 瞬态

- **仅通过** `activity` 事件分发
- **不包含在** LLM 上下文中
- **用途**：UI 实时反馈
- **类型**：thinking（思考中）、executing（执行中）、executed（已执行）、retrying（重试中）、error（错误）

## 🧩 Prompt 组装结构

发送给 LLM 的完整消息结构：

```
messages = [
    {
        role: "system",
        content: "<system_prompt.md 的内容>"
    },
    {
        role: "user",
        content: "
            <instructions>（可选，系统/页面级指令）</instructions>

            <agent_state>
                <user_request>点击登录按钮</user_request>
                <step_info>Step 1 of 40 max possible steps</step_info>
            </agent_state>

            <agent_history>
                <step_1>上一步的反思和结果</step_1>
                <sys>观察信息</sys>
            </agent_history>

            <browser_state>
                Page info: 1920x1080px viewport...
                [0]<button>登录</button>
                [1]<input placeholder='用户名'/>
                [2]<input type='password'/>
            </browser_state>
        "
    }
]

tools = {
    AgentOutput: {
        inputSchema: {
            evaluation_previous_goal: string,
            memory: string,
            next_goal: string,
            action: click | input | scroll | done | ...
        }
    }
}
```
