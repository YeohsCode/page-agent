# PageAgent 核心模块详解

## 1. 📦 `packages/page-agent` — 主入口包

**作用**：用户直接使用的入口，合并了所有子模块，提供带 UI 的完整 Agent。

### 核心文件

| 文件 | 作用 |
|------|------|
| `src/PageAgent.ts` | 主类，继承 `PageAgentCore`，自动创建 `PageController` 和 `Panel` |
| `src/demo.ts` | IIFE 演示入口，可通过 `<script>` 标签一行代码加载 |

### `PageAgent` 类（仅 29 行代码！）

```typescript
export class PageAgent extends PageAgentCore {
    panel: Panel  // UI 控制面板

    constructor(config: PageAgentConfig) {
        // 1. 创建页面控制器（默认开启 mask 遮罩）
        const pageController = new PageController({
            ...config,
            enableMask: config.enableMask ?? true,
        })
        // 2. 初始化核心 Agent
        super({ ...config, pageController })
        // 3. 创建 UI 面板
        this.panel = new Panel(this, { language: config.language })
    }
}
```

**初学者要点**：`PageAgent` 本身非常薄，它只是一个"组装器"，把 Core + PageController + Panel 组合在一起。

## 2. 🧠 `packages/core` — 核心 Agent 引擎

**作用**：实现 ReAct 循环的核心逻辑，不包含 UI。这是整个项目最重要的模块。

### 核心文件

| 文件 | 作用 |
|------|------|
| `src/PageAgentCore.ts` | ⭐ 核心 Agent 类，实现 ReAct 循环 |
| `src/types.ts` | 所有类型定义 |
| `src/tools/index.ts` | 内置工具定义（click, input, scroll 等） |
| `src/prompts/system_prompt.md` | 系统提示词模板 |
| `src/utils/autoFixer.ts` | 自动修复 LLM 响应格式 |

### `PageAgentCore.execute()` 方法 — 核心循环

```
execute(task) 的执行流程：

1. 初始化状态（重置历史记录、显示遮罩）
2. while (true) 循环：
   a. 观察阶段：获取浏览器状态（getBrowserState）
   b. 处理观察事件（URL 变化检测、步数警告等）
   c. 组装 Prompt（系统提示 + 用户提示 + 历史记录 + 浏览器状态）
   d. 调用 LLM（思考阶段）
   e. 执行 LLM 返回的动作（行动阶段）
   f. 记录历史
   g. 如果动作是 "done" → 结束循环
   h. 步数超限 → 结束循环
   i. 等待 stepDelay → 下一步
```

### 事件系统

`PageAgentCore` 继承了 `EventTarget`，支持以下事件：

| 事件 | 说明 |
|------|------|
| `statuschange` | 状态变化（idle → running → completed/error） |
| `historychange` | 历史记录更新 |
| `activity` | 实时活动反馈（thinking, executing 等） |
| `dispose` | Agent 销毁 |

### MacroTool（宏工具）设计

一个非常巧妙的设计：所有工具被合并成一个"宏工具" `AgentOutput`，强制 LLM 在每次输出时都遵循"反思后行动"模式：

```json
{
  "evaluation_previous_goal": "上一步的评估",
  "memory": "需要记住的信息",
  "next_goal": "下一步的目标",
  "action": {
    "click_element_by_index": { "index": 5 }
  }
}
```

## 3. 🤖 `packages/llms` — LLM 通信模块

**作用**：负责与各种 LLM API 通信，采用 OpenAI 兼容格式。

### 核心文件

| 文件 | 作用 |
|------|------|
| `src/index.ts` | `LLM` 类，带重试逻辑 |
| `src/OpenAIClient.ts` | OpenAI 兼容的 HTTP 客户端 |
| `src/types.ts` | 类型定义（Message, Tool, LLMConfig 等） |
| `src/utils.ts` | Zod→OpenAI 工具转换、模型适配补丁 |
| `src/errors.ts` | 错误类型定义 |

### LLM 调用流程

```
1. LLM.invoke(messages, tools, signal)
   ↓
2. withRetry() 包裹（自动重试）
   ↓
3. OpenAIClient.invoke()
   a. 将 Zod Schema 转为 OpenAI Tool 格式
   b. 应用模型补丁（Qwen/Claude/GPT/Gemini 等）
   c. 发送 HTTP POST 到 baseURL/chat/completions
   d. 解析响应，提取 tool_call
   e. 用 Zod Schema 验证参数
   f. 执行工具函数
   g. 返回结果
```

### 支持的模型

通过 `modelPatch()` 函数，自动适配不同模型的参数差异：

| 模型 | 适配内容 |
|------|----------|
| Qwen | 温度调整、关闭 thinking |
| Claude | tool_choice 格式转换、关闭 thinking |
| GPT | reasoning_effort 设置 |
| Gemini | reasoning_effort 设置 |
| Grok | 移除 tool_choice |
| MiniMax | 温度区间修正 |

### 错误类型

| 类型 | 是否重试 | 说明 |
|------|----------|------|
| `network_error` | ✅ | 网络错误 |
| `rate_limit` | ✅ | 请求限流 |
| `server_error` | ✅ | 服务器 5xx |
| `no_tool_call` | ✅ | 模型没有调用工具 |
| `invalid_tool_args` | ✅ | 参数格式错误 |
| `auth_error` | ❌ | 认证失败 |
| `context_length` | ❌ | 上下文太长 |
| `content_filter` | ❌ | 内容被过滤 |

## 4. 🎮 `packages/page-controller` — 页面控制器

**作用**：负责所有 DOM 操作，提供与 LLM 无关的页面交互能力。

### 核心文件

| 文件 | 作用 |
|------|------|
| `src/PageController.ts` | 主控制器类 |
| `src/actions.ts` | 具体的元素交互实现 |
| `src/dom/` | DOM 提取和简化引擎 |
| `src/mask/` | 可视化遮罩（阻止用户干预） |

### DOM 处理管线

```
真实 DOM
  ↓ getFlatTree()
FlatDomTree（扁平化 DOM 树）
  ↓ flatTreeToString()
简化 HTML 文本（给 LLM 阅读）
  ↓ getSelectorMap()
索引映射表（index → Element）
```

简化后的 HTML 示例：
```
[0]<button>登录</button>
[1]<input placeholder="搜索..."/>
[2]<a href="/products">商品列表</a>
```

### 支持的操作

| 方法 | 功能 |
|------|------|
| `clickElement(index)` | 点击元素（模拟完整鼠标事件序列） |
| `inputText(index, text)` | 输入文本（支持 input/textarea/contenteditable） |
| `selectOption(index, text)` | 选择下拉选项 |
| `scroll(options)` | 垂直滚动 |
| `scrollHorizontally(options)` | 水平滚动 |
| `executeJavascript(script)` | 执行 JS 代码（实验性） |
| `getBrowserState()` | 获取完整浏览器状态 |

### 点击实现细节（actions.ts）

模拟真实用户的完整事件序列：
```
scrollIntoView → movePointerToElement → mouseenter → mouseover 
→ mousedown → focus → mouseup → click
```

## 5. 🎨 `packages/ui` — UI 面板

**作用**：提供可嵌入网页的控制面板。

### 核心文件

| 文件 | 作用 |
|------|------|
| `src/panel/Panel.ts` | 面板类（纯 DOM 构建，无框架） |
| `src/panel/Panel.module.css` | 面板样式 |
| `src/i18n/` | 国际化（中/英文） |

### Panel 功能

- **任务输入框**：用户可以在面板中直接输入任务
- **状态指示器**：显示当前 Agent 状态（思考中/执行中/完成/错误）
- **历史记录**：显示每一步的反思和行动
- **停止/关闭按钮**：可中断任务或关闭面板
- **Ask User 交互**：Agent 可以向用户提问并等待回答

### 与 Agent 解耦

Panel 通过 `PanelAgentAdapter` 接口与 Agent 通信，不直接依赖 `PageAgentCore`。这意味着你可以用自己的 Agent 实现来驱动 Panel。

## 6. 🔌 `packages/mcp` — MCP 服务

**作用**：让 Claude Desktop、Copilot 等 AI 客户端通过 MCP 协议控制浏览器。

### 架构

```
Claude Desktop ←(stdio/MCP)→ MCP Server ←(WebSocket)→ Hub Tab ←→ 浏览器页面
```

### MCP 工具

| 工具 | 说明 |
|------|------|
| `execute_task` | 执行浏览器自动化任务（阻塞直到完成） |
| `get_status` | 获取连接和 Agent 状态 |
| `stop_task` | 停止当前任务 |
