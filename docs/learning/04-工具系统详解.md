# PageAgent 工具系统详解

## 🔧 什么是"工具"？

在 PageAgent 中，"工具"（Tool）是 Agent 可以执行的具体操作。LLM 通过选择合适的工具来与网页交互。

每个工具由三部分组成：

```typescript
interface PageAgentTool {
    description: string                    // 工具描述（给 LLM 看的）
    inputSchema: z.ZodType                 // 参数 Schema（用 Zod 定义）
    execute: (args) => Promise<string>     // 执行函数
}
```

## 📋 内置工具一览

### 1. `click_element_by_index` — 点击元素

```typescript
// 描述：Click element by index
// 参数：
{
    index: number  // 元素索引（从简化 HTML 中获取）
}
// 示例：LLM 看到 [5]<button>提交</button>，返回 { index: 5 }
```

### 2. `input_text` — 输入文本

```typescript
// 描述：Click and type text into an interactive input element
// 参数：
{
    index: number,  // 元素索引
    text: string    // 要输入的文本
}
// 注意：会自动先点击元素获取焦点，再输入文本
```

### 3. `select_dropdown_option` — 选择下拉选项

```typescript
// 描述：Select dropdown option by the text of the option
// 参数：
{
    index: number,  // <select> 元素的索引
    text: string    // 选项的文本内容
}
```

### 4. `scroll` — 垂直滚动

```typescript
// 描述：Scroll the page vertically
// 参数：
{
    down: boolean,           // true=向下, false=向上
    num_pages?: number,      // 滚动页数（默认 0.1，即 10% 视口高度）
    pixels?: number,         // 精确像素数（优先于 num_pages）
    index?: number           // 可选：滚动特定元素而非页面
}
```

### 5. `scroll_horizontally` — 水平滚动

```typescript
// 描述：Scroll the page horizontally
// 参数：
{
    right: boolean,    // true=向右, false=向左
    pixels: number,    // 滚动像素数
    index?: number     // 可选：滚动特定元素
}
```

### 6. `wait` — 等待

```typescript
// 描述：Wait for x seconds
// 参数：
{
    seconds: number  // 等待秒数（1-10）
}
// 注意：会自动减去 LLM 调用已花费的时间
```

### 7. `ask_user` — 询问用户

```typescript
// 描述：Ask the user a question and wait for their answer
// 参数：
{
    question: string  // 要问用户的问题
}
// 注意：仅在设置了 onAskUser 回调时可用
```

### 8. `done` — 完成任务

```typescript
// 描述：Complete task
// 参数：
{
    text: string,        // 最终回复给用户的文本
    success: boolean     // 任务是否成功（默认 true）
}
// 注意：这是唯一能结束 ReAct 循环的工具
```

### 9. `execute_javascript` — 执行 JS（实验性）

```typescript
// 描述：Execute JavaScript code on the current page
// 参数：
{
    script: string  // 要执行的 JS 代码
}
// ⚠️ 需要主动开启：experimentalScriptExecutionTool: true
```

## 🔨 MacroTool 设计模式

### 为什么不直接用多个工具？

传统 LLM 工具调用中，模型会选择一个工具并返回参数。但 PageAgent 需要 LLM 在每一步都进行**反思**（评估上一步、更新记忆、规划下一步），所以采用了 MacroTool 模式：

```
普通模式：LLM → 选择工具 → 执行

MacroTool：LLM → 强制输出反思 + 选择工具 → 执行
```

### MacroTool 结构

```typescript
{
    // 反思部分（强制 LLM 思考）
    evaluation_previous_goal?: string,  // 评估上一步
    memory?: string,                    // 短期记忆
    next_goal?: string,                 // 下一步目标

    // 行动部分（从所有工具中选一个）
    action: {
        "click_element_by_index": { index: 5 }
        // 或 "input_text": { index: 3, text: "hello" }
        // 或 "done": { text: "完成", success: true }
        // ... 只能选一个
    }
}
```

这种设计用 Zod 的 `z.union` 实现，编译为 JSON Schema 后发送给 LLM。

## 🛠️ 自定义工具

你可以添加、覆盖或删除工具：

```typescript
import { z } from 'zod/v4'
import { tool, PageAgent } from 'page-agent'

const agent = new PageAgent({
    model: 'qwen3.5-plus',
    baseURL: 'https://...',
    apiKey: 'sk-...',
    customTools: {
        // 添加新工具
        my_tool: tool({
            description: '我的自定义工具',
            inputSchema: z.object({
                param: z.string(),
            }),
            execute: async function(this: PageAgent, input) {
                // this 指向 PageAgent 实例
                // 可以访问 this.pageController 等
                return `工具执行结果: ${input.param}`
            },
        }),

        // 覆盖已有工具
        ask_user: tool({
            description: '自定义的询问用户工具',
            inputSchema: z.object({ question: z.string() }),
            execute: async function(input) {
                // 你的自定义实现
                return '用户回答: ...'
            },
        }),

        // 删除工具（设为 null）
        execute_javascript: null,  // 禁用 JS 执行
    },
})
```

## 🔧 autoFixer — 自动修复 LLM 响应

不同的 LLM 可能返回各种不标准的格式，`autoFixer` 模块会自动修复：

| 问题 | 修复方式 |
|------|----------|
| 没有 tool_calls 但在 content 中返回 JSON | 提取 JSON 并构造 tool_call |
| 返回具体工具名而非 `AgentOutput` | 包装成 `{ action: ... }` |
| 参数被双重 JSON 序列化 | 递归解析 |
| 嵌套的 function call 格式 | 展开嵌套 |
| 缺少 action 字段 | 默认 `wait 1s` |
| 基本类型参数（如 `{ "click": 2 }`） | 智能推断成 `{ "click": { "index": 2 } }` |
