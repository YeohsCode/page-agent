# PageAgent 初学者教学入门指南 — 项目概览

## 📌 这个项目是什么？

**PageAgent** 是阿里巴巴开源的一个 **网页 AI Agent（智能代理）**。简单来说，它可以让你用**自然语言**来控制网页上的各种操作，比如：

- "点击登录按钮"
- "在搜索框输入 iPhone 16"
- "把页面滚动到底部"

它直接运行在你的网页里（浏览器端），不需要 Python、无头浏览器或浏览器扩展（基本功能下）。

## 🏗️ 项目整体架构

PageAgent 是一个 **monorepo（单仓多包）** 项目，使用 npm workspaces 管理。它包含以下核心包：

```
packages/
├── page-agent/        # 🎯 主入口包 - 带 UI 面板，npm 发布为 "page-agent"
├── core/              # 🧠 核心逻辑 - 无 UI 的 Agent 引擎，npm: "@page-agent/core"
├── llms/              # 🤖 LLM 客户端 - 与大语言模型通信，npm: "@page-agent/llms"
├── page-controller/   # 🎮 页面控制器 - DOM 操作和视觉反馈，npm: "@page-agent/page-controller"
├── ui/                # 🎨 UI 面板 - 控制面板和国际化，npm: "@page-agent/ui"
├── mcp/               # 🔌 MCP 服务 - 让 Claude Desktop 等 AI 客户端控制浏览器
├── extension/         # 🧩 Chrome 扩展 - 跨页面任务支持（开发中）
└── website/           # 🌐 文档网站 - Demo 和文档页面
```

## 🔄 核心工作流程

PageAgent 采用 **ReAct (Reasoning + Acting) 循环模式**：

```
用户输入任务 → 观察网页状态 → LLM 思考 → 执行动作 → 观察结果 → 继续循环...
```

一次完整的步骤如下：

1. **观察（Observe）**：提取当前页面的 DOM 信息（按钮、输入框等交互元素）
2. **思考（Think）**：将页面信息和任务发送给 LLM，LLM 进行"反思后行动"
3. **行动（Act）**：根据 LLM 的指令，执行点击、输入等操作
4. **循环（Loop）**：回到步骤 1，直到任务完成或达到最大步数

## 📦 各模块依赖关系

```
page-agent（主入口，带 UI）
    ├── core（核心 Agent 逻辑）
    │     ├── llms（LLM 通信）
    │     └── page-controller（DOM 操作）
    └── ui（面板界面）
```

- **page-agent** 依赖所有其他包，是用户直接使用的入口
- **core** 是核心引擎，不包含 UI
- **llms** 只负责与 LLM API 通信，不依赖其他内部包
- **page-controller** 只负责 DOM 操作，不依赖 LLM
- **ui** 只负责面板显示，通过接口与 Agent 解耦
