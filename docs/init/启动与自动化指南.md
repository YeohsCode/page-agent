# PageAgent 启动与自动化指南

## 📌 核心问题回答

### Q: PageAgent 是否依赖 LLM 来实现网页自动化？

**是的，PageAgent 100% 依赖 LLM。**

### Q: 是第一次还是每次运行都需要 LLM？

**每次运行都需要 LLM。** 详细说明：

| 方面 | 说明 |
|------|------|
| **何时调用 LLM** | 每一步（step）都调用一次 LLM API |
| **一个任务多少步** | 通常 3-20 步（最大可配置，默认 40 步） |
| **是否有离线模式** | ❌ 没有，没有"录制回放"功能 |
| **能否缓存结果** | ❌ 不能，因为每次页面状态不同 |

### 工作原理

PageAgent 采用 **ReAct（推理+行动）循环**：

```
每一步都是：
  1. 提取当前页面的 DOM 信息（文本形式，不用截图）
  2. 将页面信息 + 任务 + 历史记录发送给 LLM
  3. LLM 返回：反思 + 下一步动作（点击/输入/滚动等）
  4. 执行动作
  5. 回到步骤 1
```

**关键特点**：
- 不使用截图/多模态 — 只用文本化的 DOM
- 不使用 Python/Selenium — 纯前端 JavaScript
- 不需要无头浏览器 — 直接在页面内运行
- 支持任何 OpenAI 兼容的 LLM（Qwen、GPT、Claude、Ollama 等）

## 🚀 Step by Step：启动项目并实现网页自动化

### 方式一：最快体验（一行代码，无需安装）

> 适合：快速体验，无需安装任何东西

#### 步骤 1：打开任意网页

在浏览器中打开你想要自动化的网页，比如 `https://www.baidu.com`

#### 步骤 2：打开浏览器控制台

按 `F12` 或 `Cmd+Option+I`（macOS）打开开发者工具，切换到 Console 标签。

#### 步骤 3：注入 PageAgent 脚本

在控制台中粘贴并执行：

```javascript
var s = document.createElement('script');
s.src = 'https://cdn.jsdelivr.net/npm/page-agent@1.6.0/dist/iife/page-agent.demo.js';
s.crossOrigin = 'true';
document.head.appendChild(s);
```

#### 步骤 4：使用 Agent

页面底部会出现一个控制面板，输入自然语言指令即可：
- "搜索 PageAgent"
- "点击第一个搜索结果"

> ⚠️ 这种方式使用的是 PageAgent 提供的**免费测试 API**，仅供技术评估，不要用于生产环境。

### 方式二：NPM 安装（推荐用于开发）

> 适合：在自己的项目中集成 PageAgent

#### 步骤 1：确认环境

```bash
# 检查 Node.js 版本（需要 >= 20）
node --version

# 如果没安装或版本太低：
brew install node@22
```

#### 步骤 2：创建项目（如果还没有）

```bash
mkdir my-page-agent-demo
cd my-page-agent-demo
npm init -y
```

#### 步骤 3：安装 PageAgent

```bash
npm install page-agent
```

#### 步骤 4：获取 LLM API Key

你需要一个支持 OpenAI 兼容接口的 LLM API。以下是几个选择：

**选项 A：阿里云·百炼（推荐，有免费额度）**
1. 访问 https://dashscope.console.aliyun.com/
2. 注册并获取 API Key
3. 使用模型 `qwen3.5-plus`

**选项 B：OpenAI**
1. 访问 https://platform.openai.com/
2. 获取 API Key
3. 使用模型 `gpt-5.2` 或 `gpt-5-mini`

**选项 C：本地 Ollama（免费，无需联网）**
1. 安装 Ollama：`brew install ollama`
2. 拉取模型：`ollama pull qwen3:14b`
3. 启动服务：`ollama serve`
4. baseURL: `http://localhost:11434/v1`

#### 步骤 5：在你的网页中使用

创建 `index.html`：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>PageAgent Demo</title>
</head>
<body>
    <h1>我的网页</h1>
    <button id="btn1">按钮1</button>
    <button id="btn2">按钮2</button>
    <input type="text" placeholder="输入框" />

    <script type="module">
        import { PageAgent } from 'page-agent'

        // 创建 Agent 实例
        const agent = new PageAgent({
            // === LLM 配置（必填）===
            model: 'qwen3.5-plus',
            baseURL: 'https://dashscope.aliyuncs.com/compatible-mode/v1',
            apiKey: 'sk-你的API密钥',

            // === 可选配置 ===
            language: 'zh-CN',     // Agent 使用中文
            maxSteps: 20,          // 最大步数
            enableMask: true,      // 执行时显示遮罩
        })

        // 显示控制面板
        agent.panel.show()

        // 你也可以通过代码直接执行任务
        // const result = await agent.execute('点击按钮1，然后在输入框中输入 Hello')
        // console.log(result)  // { success: true, data: '...', history: [...] }
    </script>
</body>
</html>
```

#### 步骤 6：启动开发服务器

```bash
# 使用 Vite 启动（推荐）
npx vite

# 或者使用任何静态服务器
npx serve .
```

#### 步骤 7：在浏览器中测试

1. 打开 `http://localhost:5173`（Vite 默认端口）
2. 你会看到底部的 PageAgent 控制面板
3. 输入任务，如："点击按钮1"
4. 观察 Agent 自动执行操作

## 📝 编程方式使用详解

### 基本用法

```typescript
import { PageAgent } from 'page-agent'

const agent = new PageAgent({
    model: 'qwen3.5-plus',
    baseURL: 'https://dashscope.aliyuncs.com/compatible-mode/v1',
    apiKey: 'sk-xxx',
})

// 执行任务（返回 Promise）
const result = await agent.execute('在搜索框输入 PageAgent 并点击搜索')

console.log(result.success)   // true/false
console.log(result.data)      // Agent 的最终回复文本
console.log(result.history)   // 完整的步骤历史记录
```

### 监听事件

```typescript
// 状态变化
agent.addEventListener('statuschange', () => {
    console.log('状态:', agent.status) // idle/running/completed/error
})

// 实时活动
agent.addEventListener('activity', (e) => {
    const activity = e.detail
    switch (activity.type) {
        case 'thinking':
            console.log('🧠 思考中...')
            break
        case 'executing':
            console.log(`🔨 执行: ${activity.tool}`)
            break
        case 'executed':
            console.log(`✅ 完成: ${activity.output}`)
            break
    }
})
```

### 使用生命周期钩子

```typescript
const agent = new PageAgent({
    model: 'qwen3.5-plus',
    baseURL: '...',
    apiKey: '...',

    onBeforeStep: async (agent, step) => {
        console.log(`即将执行第 ${step} 步`)
    },
    onAfterStep: async (agent, history) => {
        console.log(`第 ${history.length} 步完成`)
    },
    onBeforeTask: async (agent) => {
        console.log('任务开始')
    },
    onAfterTask: async (agent, result) => {
        console.log('任务结束:', result.success)
    },
})
```

### 手动停止任务

```typescript
// 随时可以停止
agent.stop()
```

### 销毁 Agent

```typescript
// 清理资源
agent.dispose()
// 之后不能再使用这个实例
```

## 🛠 本地开发与调试 (Local Debug)

> 适合：想要修改源代码、解决 Bug 或开发新功能的开发者。

### 1. 核心构建流程

在 monorepo 中，由于包之间存在相互依赖，必须先构建所有库：

```bash
# 安装依赖
npm install

# 构建所有子包的 dist (生成 .d.ts 和编译后的 JS)
npm run build:libs
```

### 2. 启动开发服务器

你可以根据调试目标选择不同的模式：

#### 方案 A：调试文档与集成示例 (Website)
最通用的方式，启动一个包含文档和交互式沙盒的网站。
```bash
npm start
# 访问 http://localhost:5173 
```

#### 方案 B：调试 Agent 核心功能 (Demo)
如果你想在**真实的第三方网站**上调试 Agent，建议使用此模式。
```bash
npm run dev:demo
# 访问 http://localhost:5174/page-agent.demo.js (IIFE 脚本)
```
配合书签（见方式四）即可在任意页面加载你正在修改的代码。

### 3. 调试技巧

#### A. 断点调试
1. 在浏览器开发者工具 (F12) 的 `Sources` 面板中。
2. 找到 `top -> page-agent.demo.js` (如果是 IIFE 模式) 或对应的 Webpack/Vite 源码映射。
3. 可以在 `packages/core/src/PageAgentCore.ts` 等源码中直接打断点。

#### B. 查看 Agent 原始输出
在 `packages/core/src/PageAgentCore.ts` 中，Agent 的所有状态变化和 LLM 交互都会通过 `activity` 事件发出。你可以在控制台中查看：
```javascript
agent.addEventListener('activity', e => console.log('DEBUG:', e.detail));
```

#### C. 环境变量调试
确保根目录下的 `.env` 文件配置正确。环境变量在构建时会被注入到 demo 脚本中：
- `LLM_API_KEY`: 你的 API Key
- `LLM_BASE_URL`: API 地址
- `LLM_MODEL_NAME`: 模型名称

### 4. 常见问题排查 (Troubleshooting)

| 问题 | 原因与解决方法 |
|------|----------------|
| **TS 报错：找不到模块** | 运行 `npm run build:libs` 生成各个包的类型定义。 |
| **修改代码没生效** | 如果是 IIFE 模式，确保 `npm run dev:demo` 正在运行（它带 --watch）。如果是库模式，需要重新构建对应的包。 |
| **LLM 报错 401/404** | 检查 `.env` 配置是否被正确加载（修改 .env 后建议重启 dev server）。 |
| **元素点击不到** | 检查 `PageController` 的简化 DOM 输出是否包含了该元素。 |

## ❓ 常见问题 (FAQ)

### Q: 为什么 Agent 执行很慢？
A: 每一步都需要调用 LLM API，取决于网络延迟和模型响应速度。可以尝试使用本地 Ollama 或调小 `stepDelay`。

### Q: 每次执行需要多少费用？
A: 取决于模型和页面复杂度。一个 5 步的简单任务约消耗 20,000-50,000 tokens。

### Q: 支持哪些浏览器？
A: 所有现代浏览器（Chrome、Firefox、Safari、Edge）。

### Q: 能跨页面操作吗？
A: 基本版只能在当前页面操作。需要跨页面请使用 Chrome 扩展版本。

### Q: 为什么选择文本 DOM 而不是截图？
A: 文本方式更高效、更便宜（不需要多模态模型）、更精确（不会有视觉错位），而且不需要特殊权限。
